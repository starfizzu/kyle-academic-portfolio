02

Validation Types: Client vs Server

⚡ Client-Side Validation

  • Runs in the browser instantly
  • Gives immediate feedback
  • Improves user experience
  • Uses HTML5 or JavaScript rules
  • Can be bypassed by the user

🔒 Server-Side Validation

  • Runs on the backend server
  • Cannot be bypassed by the user
  • Final authority for data integrity
  • Protects against malicious input
  • Always required for security
❌ Bad Error Message

✓ Good Error Message

05

Error Handling Flow

1

User Submits Input

The user fills out a form and clicks submit. The data is ready to be processed by the application.

2

Client-Side Validation Runs

The browser immediately checks the input against rules (e.g. required fields, email format). If it fails, an error message is shown instantly — no server needed.

3

Request Sent to Server

If client-side passes, the request travels to the backend server via HTTP.

4

Server-Side Validation Runs

The server re-validates the data independently. It checks format, checks the database (e.g. is this email already taken?), and verifies permissions.

5

Error Detected → Log Internally

If an error occurs (validation, auth, or system), it is logged internally with full details for debugging. Sensitive info is never sent to the client.

6

Friendly Response Returned

A structured JSON response is sent back with the correct HTTP status code and a clear, user-friendly message. No stack traces, no system details.
